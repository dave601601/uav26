# Task: ROS2 Line-Tracing + Dead-Reckoning 컨트롤러 플러그인 작성

## 컨텍스트
- 대회: 한국로봇항공기경연대회 중급부문 (실내 조난자 탐색)
- 임무: GPS 없이 바닥에 그려진 격자 구획선(폭 10cm, 셀 4m, 30×20m 영역)을
  고도 2m로 추종, 교차점에 놓인 ArUco(50×50cm) ID 식별/저장 후
  ID 오름차순으로 재방문하여 시작점 복귀, 자동 이착륙.
- 하드웨어: Intel RealSense D435 **수직 하향(downward-facing)** 1대만 사용.
- 시뮬레이터: Gazebo Harmonic (`ros-jazzy-ros-gz`).
- **PX4/MAVROS 사용 안 함.** Flight controller stack은 본 플러그인이 자체 구현.
- 빌드 환경: ROS2 Jazzy on Ubuntu 24.04 (Docker). `opencv-contrib-python`, `cv_bridge` 사용 가능.
- 워크스페이스 구조:
  ```
  /workspace/src/
    realsense-ros/        # RealSense ROS2 wrapper (이미 있음)
    world/                # Gazebo world / mesh / aruco 마커 생성 스크립트
  ```
  새 ROS2 패키지는 `/workspace/src/line_tracer/` 로 만들어라.

## 컨트롤러 아키텍처 (자작)
PX4 없이 동작하므로, 본 플러그인은 다음 두 레이어로 구성한다:

### Layer 1: Perception
- 입력: D435 컬러(or IR) 이미지 + depth 이미지 (`/camera/camera/color/image_raw`,
  `/camera/camera/aligned_depth_to_color/image_raw`, `/camera/camera/color/camera_info`)
- 처리:
  1. Canny + Probabilistic Hough Line Transform 으로 격자선 검출
  2. 가장 가까운 종/횡 라인의 중심과 이미지 중심(=드론 광축) 사이의
     **픽셀 오차 (Δu, Δv)** 와 **각도 오차 ψ_err** 계산
  3. depth 또는 고정 altitude 가정으로 픽셀 오차 → 미터 오차로 변환
     (`Δx_m = Δu * depth / fx`, `Δy_m = Δv * depth / fy`)
  4. 교차점 검출 시(가로/세로 라인 동시) waypoint 후보로 마킹
  5. ArUco(`DICT_6X6_250`) 검출하여 id + 픽셀좌표 → 미터좌표 환산하여 publish

### Layer 2: Dead-Reckoning Motion Stub
- 자작 컨트롤러가 아직 없으므로, 본 플러그인 안에 **임시 dead-reckoning 적분기**를 둔다.
- 입력 인터페이스 (외부에서 호출 가능):
  ```
  /line_tracer/pixel_error  (geometry_msgs/Vector3)   # (Δu, Δv, ψ_err) 픽셀·rad
  ```
- 변환:
  - `(Δu, Δv)` → `(Δx_m, Δy_m)` (intrinsics + 현재 추정 고도 사용)
  - 단순 P 게인으로 body-frame 속도지령 생성:
    `vx = Kp * Δx_m`, `vy = Kp * Δy_m`, `wz = Kψ * ψ_err`
  - 속도지령을 dt 간격으로 적분하여 추정 위치 (x̂, ŷ, ẑ, ψ̂) 갱신
- 출력 인터페이스 (자작 비행제어기가 나중에 구독할 토픽):
  ```
  /cmd_vel              (geometry_msgs/Twist)         # body-frame 속도지령
  /odom_dr              (nav_msgs/Odometry)           # dead-reckoning 추정 pose
  /waypoints/aruco      (visualization_msgs/MarkerArray) # 발견한 ArUco id+위치
  ```
- 고도는 D435 depth 평균(또는 ground plane fit) 기반으로 별도 z-PID stub 동작,
  목표 z = 2.0 m (파라미터화).

## 노드 구성
- `line_tracer_node` (Python or C++, Python 권장):
  - subs: color image, depth image, camera_info
  - pubs: `/line_tracer/pixel_error`, `/cmd_vel`, `/odom_dr`, `/waypoints/aruco`,
          `/line_tracer/debug_image` (Hough/aruco 오버레이)
  - 파라미터(YAML):
    `target_altitude`, `kp_xy`, `kp_yaw`, `canny_low/high`, `hough_*`,
    `aruco_dict`, `marker_size`, `dr_dt`
- 라이프사이클: takeoff → line_follow → waypoint_visit → arrange_by_id →
  return_path → land. 각 상태는 enum 으로 두고 service `/line_tracer/set_state` 로 전이.

## 비목표 (지금 하지 마라)
- ArUco 재방문 경로 최적화 알고리즘 (다음 단계)
- IMU 융합 EKF (지금은 순수 dead-reckoning, IMU 들어오면 그때 확장)
- 실제 모터 PWM 출력 — `/cmd_vel`까지만 책임지고 끝낸다

## 검증
- `ros2 launch line_tracer line_tracer.launch.py use_sim_time:=true` 가
  Gazebo Harmonic + RealSense plugin 환경에서 에러 없이 뜰 것
- `/line_tracer/debug_image` 에서 격자선과 픽셀 오차 시각화 확인
- 픽셀 오차 강제 입력(예: `ros2 topic pub /line_tracer/pixel_error ...`) 시
  `/cmd_vel` 과 `/odom_dr` 가 의도대로 변하는지 단위 테스트 케이스로 보여줄 것

먼저 패키지 골격과 line_tracer_node 의 클래스 설계, 토픽/파라미터 표를
보여주고 컨펌 받은 뒤 구현에 들어가라.
