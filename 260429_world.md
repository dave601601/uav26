## 좌표계 / 단위 합의 (line_tracer 와 맞아야 함)
- World: ENU (`world` frame)
- Body: FLU (REP-103). +x forward, +y left, +z up. `base_link` 가 드론 중심.
- Camera mount: 드론 하단 중앙, 광축 = 정확히 -Z_body (수직 하향)
  - Camera optical frame: ROS REP-103 표준 — +X right, +Y down, +Z forward
    (= 광축, 즉 -Z_body 방향)
  - 즉 world+body+optical 이 뒤집히는 회전을 정확히 SDF 에 박을 것
  - **카메라 yaw 오프셋 = 0** (드론 forward = 카메라 image v 위쪽이 되도록)
- 길이 단위: meter, 각도: radian

## 산출물

### 1. 격자 바닥 (30×20m 본 사양)
- 셀 크기 4m, 라인 폭 10cm, 영역 30m × 20m
- 출력: `src/world/mesh/grid_30x20_t0.10_cell4.stl`
- 현 `src/world/script/grid_stl.py` 확장. 파라미터: `--width 30 --depth 20 --cell 4 --line-width 0.10 --thickness 0.005`
- 색: 어두운 색 (검은 PBR material in SDF), 바닥은 밝은 회색

### 2. ArUco 마커 (DICT_6X6_250, 50×50cm)
- 크기: 0.5 × 0.5 m
- 두께: 0.001~0.005 m (얇은 plate)
- 위치: 격자 교차점 중 일부 (예: 7×5 = 35 교차점 전체 또는 5~10개 선택)
- 텍스처: `src/world/script/aruco.py` 확장 — 각 ID 의 PNG 생성, `src/world/textures/aruco_<id>.png` 저장
- SDF: 각 마커는 `<model>` 또는 `<include>`. PBR `<material>` 의 `<diffuse>` 텍스처로 PNG 사용
- 배치 사양 YAML: `src/world/config/aruco_layout.yaml`
  ```yaml
  markers:
    - id: 0
      pose: [4.0, 4.0, 0.001, 0, 0, 0]   # x y z roll pitch yaw
    - id: 1
      pose: [8.0, 4.0, 0.001, 0, 0, 0]
    ...
  ```

### 3. World SDF
- 출력: `src/world/worlds/competition.sdf`
- 포함:
  - `<physics>` Gazebo 기본
  - `<plugin filename="gz-sim-physics-system">`, `gz-sim-user-commands-system`,
    `gz-sim-scene-broadcaster-system`, `gz-sim-sensors-system` (rendering 필요)
  - 조명 (sun + 추가 directional 1~2개 — 마커 가독성)
  - ground_plane (밝은 회색 PBR)
  - 격자 mesh include
  - ArUco 마커 모델들 include (layout YAML 기반 launch 시 SDF 생성 또는 파이썬으로 SDF 빌드)

### 4. 드론 모델
- 출력: `src/world/models/uav26_quad/model.sdf` + `model.config`
- X-frame quadrotor:
  - base_link mass ~1.5 kg, inertia 적당
  - 4 rotors (앞왼/앞오/뒤왼/뒤오), 각 rotor link + joint
  - `gz-sim-multicopter-motor-model-system` 플러그인 (각 rotor 별)
- 하향 D435 카메라 sensor:
  - Color: 640×480 @ 30Hz, FOV ~69° H × 42° V
  - Depth: 640×480 @ 30Hz, range 0.3~3.0 m
  - Aligned depth (Gazebo가 직접 지원 안 하면 두 카메라 sensor 같은 frame_id 로 publish)
  - frame_id: `camera_color_optical_frame`, `camera_depth_optical_frame`
  - SDF `<sensor type="camera">` + `<sensor type="depth_camera">`
- 가짜 FC: `gz-sim-multicopter-velocity-control-system` 플러그인을 base_link 에 부착
  - subscribe: `<commandSubTopic>cmd_vel</commandSubTopic>` (gz topic, ROS 토픽 아님)
  - 안정화 PID 내장 (Gazebo 기본값으로 시작)

### 5. ros_gz 브리지
- 출력: `src/world/config/bridge.yaml`
  ```yaml
  - ros_topic_name: "/camera/camera/color/image_raw"
    gz_topic_name: "/world/competition/model/uav26_quad/link/camera_link/sensor/color/image"
    ros_type_name: "sensor_msgs/msg/Image"
    gz_type_name: "gz.msgs.Image"
    direction: GZ_TO_ROS
  - ros_topic_name: "/camera/camera/color/camera_info"
    ...
  - ros_topic_name: "/camera/camera/aligned_depth_to_color/image_raw"
    ...
  - ros_topic_name: "/cmd_vel"
    gz_topic_name: "/model/uav26_quad/cmd_vel"
    ros_type_name: "geometry_msgs/msg/Twist"
    gz_type_name: "gz.msgs.Twist"
    direction: ROS_TO_GZ
  - ros_topic_name: "/clock"
    gz_topic_name: "/clock"
    ros_type_name: "rosgraph_msgs/msg/Clock"
    gz_type_name: "gz.msgs.Clock"
    direction: GZ_TO_ROS
  ```
  (실제 gz topic 경로는 모델/센서 이름에 따라 다름 — Gazebo 띄워서 `gz topic -l` 로 확인 후 채울 것)

### 6. Launch

- 포함:
  - `IncludeLaunchDescription(ros_gz_sim/launch/gz_sim.launch.py)` with `gz_args="competition.sdf"`
  - `Node(package='ros_gz_bridge', executable='parameter_bridge', parameters=[bridge_yaml])`
  - 옵션: spawn pose 파라미터 (시작점 좌표 변경용)

## 검증
1. `colcon build --packages-select world` 통과
2. `ros2 launch world sim.launch.py` 로 Gazebo 창이 뜨고, 격자 바닥 + ArUco + 드론 보임
3. `ros2 topic list` 에 `/camera/camera/color/image_raw`, `/cmd_vel`, `/clock` 등장
4. `rqt_image_view /camera/camera/color/image_raw` 에서 격자 영상 확인
5. `ros2 topic pub --once /cmd_vel geometry_msgs/Twist '{linear: {x: 0.5}}'` 로
   드론이 +x 방향으로 0.5 m/s 정도 움직이는 것 확인
6. `ros2 launch line_tracer line_tracer.launch.py use_sim_time:=true` 와 함께 실행 시,
   line_tracer 가 카메라 토픽을 받고 `/odom_dr` 발행하는지 확인
